Function is_authoritative 

pub fn is_authoritative(rec: &TaskRecord, current_boot_id: &str) -> bool
Expand description

True iff this process may reconcile rec to a terminal-loss state.

Authoritative when:

  • the record is from a PRIOR boot (owner_boot_id != current_boot_id) — the daemon that owned it is gone, so its in-flight tasks are orphans; or
  • the record is same-boot but its owner_pid is no longer alive (crashed mid-run without writing a terminal state).

Never reclaims a task a live same-boot daemon is actively heart-beating.
