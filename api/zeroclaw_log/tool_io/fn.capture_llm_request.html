pub fn capture_llm_request(
    policy: LlmRequestPayloadPolicy,
    truncate_bytes: usize,
    redacted: &str,
) -> Option<ToolIoCapture>
Expand description

Capture the (already leak-scanned) LLM request payload per the request-payload policy. Unlike tool I/O there is no per-tool denylist: the whole payload is gated by the policy alone, and truncate_bytes reuses the shared tool-io truncate cap. Returns None when the policy is off (the default) so the prompt is never persisted unless opted in.

Takes the policy + cap directly (not a ResolvedPolicy) so the call site can use crate::llm_request_payload_policy without holding the full resolved bundle.
