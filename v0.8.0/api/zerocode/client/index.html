JSON-RPC 2.0 client over a local IPC stream (Unix socket / Windows named pipe, NDJSON) or WebSocket (WSS).

Wraps RpcOutbound from zeroclaw-api — the same request/response plumbing the daemon uses for bidirectional calls.
