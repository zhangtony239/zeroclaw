Expand description

Drop tools from tools that fail either gate.

  1. The parent agent’s SecurityPolicy.allowed_tools allowlist plus SecurityPolicy.excluded_tools denylist, evaluated via SecurityPolicy::is_tool_allowed.
  2. The caller-supplied caller_allowed filter (the existing agent::run-level allowed_tools parameter).

A tool survives only when BOTH gates admit its name. None on either gate is unrestricted for that gate alone. Built-in tools, MCP tools, and skill tools all flow through the same filter; the helper does not know or care about category.
