zeroclaw

El asistente de IA más rápido y pequeño.

Uso: zeroclaw [OPTIONS] <COMMAND>

Subcomandos:

  • quickstart — Quickstart — create one working agent end-to-end. Replaces the section-by-section onboarding flow with a single preset-driven path. Interactive: the flags below pre-seed checklist selectors but do not skip them; a terminal is required
  • onboard — Obsoleto. Use zeroclaw quickstart. Cualquier flag genera error
  • agent — Iniciar el bucle del agente de IA
  • gateway — Iniciar/gestionar el servidor de gateway (webhooks, websockets)
  • acp — Iniciar el servidor ACP (Protocolo de Control de Agentes) a través de stdio
  • daemon — Iniciar el tiempo de ejecución autónomo de larga duración (gateway + canales + latido + programador)
  • service — Administrar el ciclo de vida del servicio del sistema operativo (servicio de usuario de launchd/systemd)
  • doctor — Ejecuta diagnósticos para la frescura del daemon/programador/canal
  • status — Mostrar el estado del sistema (detalles completos)
  • estop — Activar, inspeccionar y reanudar los estados de parada de emergencia
  • cron — Configurar y gestionar tareas programadas
  • models — Gestionar los catálogos de modelos de model_provider
  • providers — List supported AI model providers, or manage provider aliases
  • channel — Administrar canales (Telegram, Discord, Slack)
  • agents — Manage agent aliases (create/list/rename/delete). Distinct from agent, which runs an agent
  • channels — Manage channel aliases (create/list/rename/delete)
  • integrations — Explora más de 50 integraciones
  • skills — Administrar habilidades (capacidades definidas por el usuario)
  • browse — Explora el espacio de trabajo compartido un directorio a la vez
  • sop — Gestionar procedimientos operativos estándar (SOP)
  • migrate — Migrar datos desde otros entornos de ejecución de agentes
  • auth — Gestionar perfiles de autenticación de suscripción de model_provider
  • hardware — Descubrir e inspeccionar el hardware USB
  • peripheral — Gestiona periféricos de hardware (STM32, GPIO de RPi, etc.)
  • memory — Administrar la memoria del agente (listar, obtener, estadísticas, limpiar)
  • config — Administrar la configuración
  • update — Buscar y aplicar actualizaciones
  • self-test — Ejecutar pruebas de diagnóstico
  • completions — Genera el script de autocompletado de shell en stdout
  • desktop — Iniciar o instalar la aplicación de escritorio complementaria
  • locales — Obtener archivos de configuración regional traducidos (FTL) desde upstream

Opciones:

  • --config-dir <DIRECTORIO_CONFIG>

  • --log-level <LOG_LEVEL> — Severidad mínima registrada en la traza de tiempo de ejecución (y la capa de captura). Inmutable para el proceso. Precedencia: este flag > variable de entorno RUST_LOG > valor predeterminado por comando

    Valores posibles: error, warn, info, debug, trace

  • -v, --verbose — Muestra los registros grabados en la terminal. Desactivado por defecto: los registros van solo al archivo de traza y la terminal muestra únicamente la salida del comando. Cuando está activado, la terminal muestra eventos hasta el nivel mínimo grabado. Inmutable durante el proceso

zeroclaw quickstart

Quickstart — create one working agent end-to-end. Replaces the section-by-section onboarding flow with a single preset-driven path. Interactive: the flags below pre-seed checklist selectors but do not skip them; a terminal is required

Uso: zeroclaw quickstart [OPTIONS]

Opciones:

  • --model-provider <MODEL_PROVIDER> — Tipo de proveedor (anthropic / openai / openrouter / ollama)
  • --model <MODEL> — ID del modelo para la nueva entrada del proveedor
  • --api-key <API_KEY> — Clave de API para la nueva entrada de proveedor (omitir para ollama / local)
  • --agent <AGENT> — Alias para el nuevo agente. De forma predeterminada usa un nombre de proveedor sanitizado

zeroclaw onboard

Obsoleto. Use zeroclaw quickstart. Cualquier flag produce un error

Uso: zeroclaw onboard [COMMAND]

Subcomandos:

  • providers.models — Elige un proveedor de modelos para configurar (Anthropic, OpenAI, OpenRouter, Ollama, gateways personalizados compatibles con OpenAI, etc.). Se admiten múltiples alias por proveedor; por ejemplo, anthropic.production y anthropic.dev pueden coexistir
  • risk_profiles — Perfiles de riesgo con nombre que vinculan listas de permitidos, listas de denegados y umbrales de aprobación. Los agentes hacen referencia a uno mediante agents.<alias>.risk_profile
  • runtime_profiles — Perfiles de ajuste de tiempo de ejecución con nombre (límites de tokens, política de reintentos, tiempos de espera). Los agentes hacen referencia a uno mediante agents.<alias>.runtime_profile
  • storage — SQLite es la opción predeterminada segura para instalaciones de un solo nodo (basado en archivos, sin configuración, sin servicios adicionales). Elija Postgres para despliegues compartidos o de múltiples instancias, Qdrant para búsqueda vectorial, Markdown o Lucid para archivos legibles por humanos. Cada backend admite múltiples instancias con alias; los agentes hacen referencia a ellas mediante memory.storage_ref
  • memory — Backend de memoria persistente. SQLite es el predeterminado; selecciona none para deshabilitar por completo la recuperación a largo plazo
  • skills — Configuración de la herramienta de skills: dónde se almacenan los archivos markdown de skills en disco (por defecto, en el directorio de datos) y cómo el cargador de skills gestiona los repositorios de la comunidad. Agrega BUNDLES de skills en skill-bundles más abajo
  • skill_bundles — Paquetes con nombre de archivos de habilidades. Los agentes hacen referencia a un paquete para cargar un conjunto de capacidades al iniciar
  • mcp — Configuración de Model Context Protocol. Activa o desactiva enabled y elige carga diferida o anticipada. Los servidores MCP individuales se encuentran en mcp.servers[]
  • mcp.servers — Individual Model Context Protocol servers. Each entry binds a transport (stdio, http, sse), the command or URL to reach it, optional headers, and a tool_timeout_secs cap (≤ 600). Each server’s name is its addressable key — rename via the section page rather than editing the field directly. Group servers into bundles under mcp_bundles below
  • mcp_bundles — Named bundles of MCP servers, granted to agents that list the bundle in their mcp_bundles. Secure by default: an agent gets only the servers its bundles grant; with no bundle it gets no MCP servers
  • knowledge_bundles — Paquetes con nombre de fuentes de conocimiento (índices RAG, carpetas de documentación). Los agentes hacen referencia a un paquete para mostrar fragmentos relevantes en tiempo de inferencia
  • providers.tts: proveedores de texto a voz (OpenAI, ElevenLabs, Google, Edge, Piper). Configura uno por voz/idioma; los agentes los referencian mediante su alias
  • providers.transcription — Proveedores de conversión de voz a texto (OpenAI Whisper, Groq, Deepgram, AssemblyAI, Google, Whisper local). Configura uno por pipeline; los agentes los referencian mediante un alias
  • channels — Pick which chat platforms ZeroClaw should listen on. Global channel settings live on [channels]; each configured platform still gets its own alias
  • hardware — Opcional: periféricos de hardware (Arduino, STM32, GPIO, etc.). Omítelo si no los necesitas
  • agents — Un agente vincula un proveedor de modelos, perfiles, paquetes y canales en una única unidad despachable. Añade uno por persona; reutiliza el mismo alias en todos los canales para compartir el estado
  • peer_groups — Grupos con nombre que vinculan un canal, agentes miembros y pares externos. Aceptación mutua: dos agentes se convierten en pares solo cuando ambos aparecen en la lista agents del mismo grupo
  • cron — Tareas programadas. Cada entrada de cron vincula una expresión de programación a un prompt, un canal y un destino
  • tunnel — Opcional: expón tu gateway en internet a través de Cloudflare o ngrok. Selecciona none para mantenerlo solo en localhost
  • onboard_state — Estado del ciclo de vida del Quickstart. quickstart_completed cambia a true una vez que el Quickstart finaliza una ejecución exitosa; mientras sea false, la puerta de enlace web y la TUI inician automáticamente el Quickstart al arrancar. completed_sections es un registro heredado por sección que se conserva por compatibilidad con datos anteriores

zeroclaw agent

Iniciar el bucle del agente de IA.

Inicia una sesión de chat interactiva con el model_provider de IA configurado. Usa --message para consultas de un solo envío sin entrar en el modo interactivo.

Ejemplos: zeroclaw agent -a assistant # sesión interactiva zeroclaw agent -a assistant -m “Summarize today’s logs” # mensaje único zeroclaw agent -a assistant -p anthropic –model claude-sonnet-4-20250514 zeroclaw agent -a assistant –peripheral nucleo-f401re:/dev/ttyACM0

Uso: zeroclaw agent [OPTIONS] --agent <AGENT>

Opciones:

  • -a, --agent <AGENT> — Alias del agente configurado con el que ejecutar (debe coincidir con [agents.<alias>]). Obligatorio — no hay agente predeterminado
  • -m, --message <MESSAGE> — Modo de mensaje único (no entrar en modo interactivo)
  • --session-state-file <SESSION_STATE_FILE> — Cargar y guardar el estado de la sesión interactiva en este archivo JSON
  • -p, --model-provider <MODEL_PROVIDER> — Proveedor de modelos a utilizar (openrouter, anthropic, openai, openai-codex)
  • --model <MODEL> — Modelo a utilizar
  • -t, --temperature <TEMPERATURE> — Temperature (0.0 - 2.0, defaults to providers.models.<type>.<alias>.temperature)
  • --peripheral <PERIPHERAL> — Adjuntar un periférico (placa:ruta, por ejemplo, nucleo-f401re:/dev/ttyACM0)

zeroclaw gateway start

Inicia el servidor de puerta de enlace (webhooks, websockets).

Ejecuta la puerta de enlace HTTP/WebSocket que acepta eventos de webhook entrantes y conexiones WebSocket. La dirección de enlace predeterminada corresponde a los valores definidos en tu archivo de configuración (gateway.host / gateway.port).

Ejemplos: zeroclaw gateway start # usar valores predeterminados de configuración zeroclaw gateway start -p 8080 # escuchar en el puerto 8080 zeroclaw gateway start –host 0.0.0.0 # requiere [gateway].allow_public_bind=true o un túnel zeroclaw gateway start -p 0 # puerto aleatorio disponible

Uso: zeroclaw gateway start [OPTIONS]

Opciones:

  • -p, --port <PORT> — Puerto en el que escuchar (usar 0 para un puerto aleatorio disponible); por defecto usa la configuración gateway.port
  • --host <HOST> — Host al que vincularse; por defecto usa gateway.host de la configuración. Nota: Vincularse a 0.0.0.0 requiere gateway.allow_public_bind = true en la configuración.
  • --allow-degraded-security — Arranca incluso cuando secciones de configuración críticas para la seguridad fueron restablecidas a sus valores predeterminados durante la carga (postura debilitada). Desactivado de forma predeterminada

zeroclaw gateway get-paircode

Mostrar o generar el código de emparejamiento del gateway.

Muestra el código de emparejamiento para conectar nuevos clientes sin reiniciar la puerta de enlace. Requiere que la puerta de enlace esté en ejecución.

Con –new, genera un código de emparejamiento nuevo incluso si el gateway ya fue emparejado anteriormente (útil para agregar clientes adicionales). Esto NO revoca los tokens existentes.

Con –rotate, revoca TODOS los tokens bearer emparejados, borra el registro de dispositivos y emite un código nuevo. Use esto después de una sospecha de fuga de tokens cuando no sepa qué token fue comprometido; todos los clientes deben volver a emparejarse.

With –rotate-device ID, revokes just that device’s bearer token and issues a fresh code for re-pairing that one device.

Ejemplos: zeroclaw gateway get-paircode # mostrar el código de emparejamiento actual zeroclaw gateway get-paircode –new # agregar otro cliente (sin revocación) zeroclaw gateway get-paircode –rotate # revocar TODOS los tokens, luego emitir un código zeroclaw gateway get-paircode –rotate-device dash-1 # revocar el token de un dispositivo zeroclaw gateway get-paircode –new –port 3001 # apuntar al gateway de puerto alternativo

Uso: zeroclaw gateway get-paircode [OPTIONS]

Opciones:

  • --new — Genera un nuevo código de emparejamiento para agregar un cliente (no revoca los tokens existentes)
  • --rotate — Revoca TODOS los tokens emparejados y borra el registro de dispositivos, luego emite un nuevo código
  • --rotate-device <DEVICE_ID> — Revoca el token de portador de un solo dispositivo por id, luego emite un nuevo código
  • -p, --port <PORT> — Puerto de la puerta de enlace en ejecución que se va a consultar; el valor predeterminado es config gateway.port
  • --host <HOST> — Host de la puerta de enlace en ejecución que se va a consultar; el valor predeterminado es config gateway.host

zeroclaw acp

Inicia el servidor ACP (JSON-RPC 2.0 sobre stdio).

Inicia un servidor JSON-RPC 2.0 en stdin/stdout para la integración con IDE y herramientas. Admite la gestión de sesiones y la transmisión de respuestas del agente como notificaciones.

Métodos: initialize, session/new, session/prompt, session/stop.

Ejemplos: zeroclaw acp # iniciar el servidor ACP zeroclaw acp –max-sessions 5 # limitar las sesiones concurrentes

Configurar y gestionar tareas programadas.

Programe tareas recurrentes, de ejecución única o basadas en intervalos usando expresiones cron, marcas de tiempo RFC3339 con Z explícita o desfases horarios, duraciones o intervalos fijos.

Las expresiones cron usan el formato estándar de 5 campos: ‘min hour day month weekday’. Cuando se omite –tz, las programaciones cron usan la zona horaria local del entorno de ejecución. Para programaciones visibles para el usuario, pase –tz con una zona horaria IANA explícita.

Ejemplos: zeroclaw skills add code-review –bundle official –description “Review PRs.” zeroclaw skills add ops-runbook –description “Triage prod incidents.” –edit

Uso: zeroclaw skills add [OPTIONS] <NAME>

Argumentos:

  • <NAME> — Nombre de la skill (solo minúsculas y guiones)

Opciones:

  • --bundle <BUNDLE> — Alias del paquete de destino. Opcional cuando hay exactamente un paquete configurado
  • --description <DESCRIPTION> — Qué hace la skill y cuándo usarla (frontmatter description). Obligatorio; se solicita en TTY cuando falta
  • --license <LICENSE> — Identificador de licencia SPDX (p. ej. MIT)
  • --author <AUTHOR> — Identificador del autor de la skill
  • --version <VERSION> — versión SemVer (predeterminada: 0.1.0)
  • --category <CATEGORY> — Categoría de la skill para la agrupación en el registro
  • --no-scaffold — Omitir la creación de la estructura scripts/, references/, assets/
  • --edit — Abrir SKILL.md en $EDITOR después del scaffold

zeroclaw config

Gestionar la configuración de ZeroClaw.

Ver, establecer o inicializar propiedades de configuración mediante una ruta con puntos. Usa ‘schema’ para volcar el esquema JSON completo del archivo de configuración.

Las propiedades se direccionan mediante una ruta con puntos (por ejemplo, channels.matrix.mencionar-solo). Los campos de secretos (claves API, tokens) utilizan automáticamente una entrada enmascarada. Los campos de enumeración ofrecen una selección interactiva cuando el valor está omitido.

Ejemplos: zeroclaw config list # listar todas las propiedades zeroclaw config list –secrets # listar solo secretos zeroclaw config list –filter channels.matrix # filtrar por prefijo zeroclaw config get channels.matrix.mention-only # obtener un valor zeroclaw config set channels.matrix.mention-only true # establecer un valor zeroclaw config set channels.matrix.access-token # secreto: entrada enmascarada zeroclaw config set channels.matrix.stream-mode # enum: selección interactiva zeroclaw config init channels.matrix # inicializar sección con valores predeterminados zeroclaw config schema # imprimir JSON Schema a stdout zeroclaw config schema > schema.json

La finalización automática de rutas de propiedades se incluye automáticamente en zeroclaw completions <shell>.

Uso: zeroclaw config <COMMAND>

Subcomandos:

  • schema — Vuelca el JSON Schema completo de la configuración a stdout. Con --path, devuelve solo el fragmento del esquema para esa propiedad — el mismo payload que OPTIONS /api/config/prop?path=... devuelve por HTTP
  • list — Muestra todas las propiedades de configuración con sus valores actuales
  • get — Obtener el valor de una propiedad de configuración
  • set — Establece una propiedad de configuración (los campos secretos solicitan automáticamente una entrada enmascarada)
  • init — Inicializa las secciones no configuradas con los valores predeterminados (enabled=false)
  • migrate — Migra la configuración en disco a la versión actual del esquema (conserva los comentarios)
  • patch — Aplica un documento JSON Patch (RFC 6902) de forma atómica. Refleja PATCH /api/config
  • docs — Muestra la URL del explorador de API (más una sugerencia si el daemon no se está ejecutando)
  • generate — Genera una configuración canónica en cualquier versión de esquema compatible en stdout

zeroclaw update

Verificar y aplicar actualizaciones de ZeroClaw.

De forma predeterminada, descarga e instala la versión más reciente mediante una canalización de 6 fases: preflight, descarga, copia de seguridad, validación, cambio y prueba de humo. Realiza una reversión automática en caso de fallo.

Utiliza –check para solo verificar actualizaciones sin instalar. Usa –force para omitir el mensaje de confirmación. Usa –version para dirigirse a una versión específica en lugar de la última.

Ejemplos: zeroclaw update # descargar e instalar la última actualización de zeroclaw zeroclaw update –check # solo verificar, no instalar zeroclaw update –force # instalar sin confirmación zeroclaw update –version 0.6.0 # instalar una versión específica

Uso: zeroclaw update [OPTIONS]

Opciones:

  • --check — Solo verifica las actualizaciones, no instala
  • --force — Install even if the target is not newer (reinstall or downgrade/pin to –version)
