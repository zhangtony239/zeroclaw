zeroclaw

L’assistant IA le plus rapide et le plus léger.

Utilisation : zeroclaw [OPTIONS] <COMMAND>

Sous-commandes :

  • quickstart — Quickstart — create one working agent end-to-end. Replaces the section-by-section onboarding flow with a single preset-driven path. Interactive: the flags below pre-seed checklist selectors but do not skip them; a terminal is required
  • onboard — Obsolète. Utilisez zeroclaw quickstart. Tout indicateur provoque une erreur
  • agent — Démarrer la boucle de l’agent IA
  • gateway — Démarrer/gérer le serveur gateway (webhooks, websockets)
  • acp — Démarrer le serveur ACP (Agent Control Protocol) via stdio
  • daemon — Démarrer un runtime autonome à long terme (passerelle + canaux + battement de cœur + planificateur)
  • service — Gérer le cycle de vie du service OS (service utilisateur launchd/systemd)
  • doctor — Exécuter des diagnostics pour la fraîcheur du daemon/planificateur/chaîne
  • status — Afficher l’état du système (détails complets)
  • estop — Activer, inspecter et reprendre les états d’arrêt d’urgence
  • cron — Configurer et gérer les tâches planifiées
  • models — Gérer les catalogues de modèles model_provider
  • providers — List supported AI model providers, or manage provider aliases
  • channel — Gérer les canaux (Telegram, Discord, Slack)
  • agents — Manage agent aliases (create/list/rename/delete). Distinct from agent, which runs an agent
  • channels — Manage channel aliases (create/list/rename/delete)
  • integrations — Parcourez plus de 50 intégrations
  • skills — Gérer les compétences (capacités définies par l’utilisateur)
  • browse — Parcourir l’espace de travail partagé un répertoire à la fois
  • sop — Gérer les procédures opérationnelles standard (SOP)
  • migrate — Migrer les données depuis d’autres runtimes d’agents
  • auth — Gérer les profils d’authentification d’abonnement model_provider
  • hardware — Découvrir et analyser le matériel USB
  • peripheral — Gérer les périphériques matériels (STM32, GPIO RPi, etc.)
  • memory — Gérer la mémoire de l’agent (list, get, stats, clear)
  • config — Gérer la configuration
  • update — Vérifier et appliquer les mises à jour
  • self-test — Exécuter des tests d’automodiagnostic
  • completions — Générer le script de complétion de shell vers stdout
  • desktop — Lancer ou installer l’application de bureau associée
  • locales — Récupère les fichiers de langue traduits (FTL) depuis le dépôt amont

Options :

  • --config-dir <CONFIG_DIR>

  • --log-level <LOG_LEVEL> — Sévérité minimale enregistrée dans la trace d’exécution (et la couche de capture). Immuable pour le processus. Priorité : ce flag > variable d’environnement RUST_LOG > valeur par défaut par commande

    Valeurs possibles : error, warn, info, debug, trace

  • -v, --verbose — Affiche les journaux enregistrés dans le terminal. Désactivé par défaut : les journaux sont écrits uniquement dans le fichier de trace et le terminal n’affiche que la sortie des commandes. Lorsqu’il est activé, le terminal affiche les événements jusqu’au niveau plancher enregistré. Immuable pour le processus

zeroclaw quickstart

Quickstart — create one working agent end-to-end. Replaces the section-by-section onboarding flow with a single preset-driven path. Interactive: the flags below pre-seed checklist selectors but do not skip them; a terminal is required

Utilisation : zeroclaw quickstart [OPTIONS]

Options :

  • --model-provider <MODEL_PROVIDER> — Type de fournisseur (anthropic / openai / openrouter / ollama)
  • --model <MODEL> — Identifiant du modèle pour la nouvelle entrée de fournisseur
  • --api-key <API_KEY> — Clé API pour la nouvelle entrée de fournisseur (à omettre pour ollama / local)

zeroclaw onboard

Obsolète. Utilisez zeroclaw quickstart. Toute option provoque une erreur

Utilisation : zeroclaw onboard [COMMAND]

Sous-commandes :

  • providers.models — Choisissez un fournisseur de modèles à configurer (Anthropic, OpenAI, OpenRouter, Ollama, passerelles personnalisées compatibles OpenAI, etc.). Plusieurs alias par fournisseur sont pris en charge — par exemple, anthropic.production et anthropic.dev peuvent coexister
  • risk_profiles — Profils de risque nommés associant les listes d’autorisation, les listes de blocage et les seuils d’approbation. Les agents en référencent un via agents.<alias>.risk_profile
  • runtime_profiles — Profils nommés de réglage d’exécution (limites de tokens, politique de nouvelles tentatives, délais d’expiration). Les agents en référencent un via agents.<alias>.runtime_profile
  • storage — SQLite est le choix sûr par défaut pour les installations à nœud unique (basé sur fichiers, zéro configuration, aucun service supplémentaire). Choisissez Postgres pour les déploiements partagés ou multi-instances, Qdrant pour la recherche vectorielle, Markdown ou Lucid pour des fichiers lisibles par l’humain. Chaque backend prend en charge plusieurs instances avec alias ; les agents les référencent via memory.storage_ref
  • memory — Backend de mémoire persistante. SQLite est la valeur par défaut ; choisissez none pour désactiver entièrement la mémoire à long terme
  • skills — Paramètres de l’outil Skills — emplacement des fichiers markdown des skills sur le disque (par défaut, le répertoire de données), et comment le chargeur de skills gère les dépôts communautaires. Ajoutez des BUNDLES de skills sous skill-bundles ci-dessous
  • skill_bundles — Ensembles nommés de fichiers de compétences. Les agents référencent un ensemble pour charger un groupe de capacités au démarrage
  • mcp — Paramètres du Model Context Protocol. Activez/désactivez enabled et choisissez le chargement différé ou immédiat. Les serveurs MCP individuels se trouvent sous mcp.servers[]
  • mcp.servers — Individual Model Context Protocol servers. Each entry binds a transport (stdio, http, sse), the command or URL to reach it, optional headers, and a tool_timeout_secs cap (≤ 600). Each server’s name is its addressable key — rename via the section page rather than editing the field directly. Group servers into bundles under mcp_bundles below
  • mcp_bundles — Named bundles of MCP servers, granted to agents that list the bundle in their mcp_bundles. Secure by default: an agent gets only the servers its bundles grant; with no bundle it gets no MCP servers
  • knowledge_bundles — Lots nommés de sources de connaissances (index RAG, dossiers de documentation). Les agents référencent un lot pour faire remonter les extraits pertinents au moment de l’inférence
  • providers.tts — Fournisseurs de synthèse vocale (OpenAI, ElevenLabs, Google, Edge, Piper). Configurez-en un par voix / langue ; les agents les référencent par alias
  • providers.transcription — Fournisseurs de reconnaissance vocale (OpenAI Whisper, Groq, Deepgram, AssemblyAI, Google, Whisper local). Configurez-en un par pipeline ; les agents y font référence par alias
  • channels — Pick which chat platforms ZeroClaw should listen on. Global channel settings live on [channels]; each configured platform still gets its own alias
  • hardware — Optionnel : périphériques matériels (Arduino, STM32, GPIO, etc.). Ignorez si vous n’en avez pas besoin
  • agents — Un agent associe un fournisseur de modèle, des profils, des bundles et des canaux en une seule unité distribuable. Ajoutez-en un par persona ; réutilisez le même alias entre les canaux pour partager l’état
  • peer_groups — Groupes nommés liant un canal, des agents membres et des pairs externes. Adhésion mutuelle : deux agents deviennent pairs uniquement lorsque les deux figurent dans la liste agents du même groupe
  • cron — Tâches planifiées. Chaque entrée cron associe une expression de planification à une invite, un canal et une cible
  • tunnel — Optionnel : expose votre passerelle sur Internet via Cloudflare ou ngrok. Choisissez none pour la garder uniquement en localhost
  • onboard_state — État du cycle de vie du Quickstart. quickstart_completed passe à true une fois que le Quickstart termine une exécution réussie ; tant qu’il est à false, la passerelle web et la TUI lancent automatiquement le Quickstart au démarrage. completed_sections est un registre par section hérité, conservé pour la rétrocompatibilité avec les données antérieures

zeroclaw agent

Démarrer la boucle de l’agent IA.

Lance une session de chat interactive avec le model_provider d’IA configuré. Utilisez –message pour des requêtes uniques sans entrer en mode interactif.

Exemples : zeroclaw agent -a assistant # session interactive zeroclaw agent -a assistant -m “Summarize today’s logs” # message unique zeroclaw agent -a assistant -p anthropic –model claude-sonnet-4-20250514 zeroclaw agent -a assistant –peripheral nucleo-f401re:/dev/ttyACM0

Utilisation : zeroclaw agent [OPTIONS] --agent <AGENT>

Options :

  • -a, --agent <AGENT> — Alias d’agent configuré sous lequel s’exécuter (doit correspondre à [agents.<alias>]). Obligatoire — il n’y a pas d’agent par défaut
  • -m, --message <MESSAGE> — Mode de message unique (ne pas entrer en mode interactif)
  • --session-state-file <SESSION_STATE_FILE> — Charger et enregistrer l’état de la session interactive dans ce fichier JSON
  • -p, --model-provider <MODEL_PROVIDER> — Fournisseur de modèle à utiliser (openrouter, anthropic, openai, openai-codex)
  • --model <MODEL> — Modèle à utiliser
  • -t, --temperature <TEMPERATURE> — Temperature (0.0 - 2.0, defaults to providers.models.<type>.<alias>.temperature)
  • --peripheral <PÉRIPHÉRIQUE> — Attacher un périphérique (carte:chemin, par exemple nucleo-f401re:/dev/ttyACM0)

zeroclaw gateway start

Démarrer le serveur de passerelle (webhooks, websockets).

Exécute la passerelle HTTP/WebSocket qui accepte les événements webhook entrants et les connexions WebSocket. L’adresse de liaison par défaut correspond aux valeurs définies dans votre fichier de configuration (gateway.host / gateway.port).

Exemples : zeroclaw gateway start # utiliser les valeurs par défaut de la configuration zeroclaw gateway start -p 8080 # écouter sur le port 8080 zeroclaw gateway start –host 0.0.0.0 # nécessite [gateway].allow_public_bind=true ou un tunnel zeroclaw gateway start -p 0 # port aléatoire disponible

Utilisation : zeroclaw gateway start [OPTIONS]

Options :

  • -p, --port <PORT> — Port d’écoute (utilisez 0 pour un port aléatoire disponible) ; par défaut, utilise la valeur de gateway.port dans la configuration.
  • --host <HOST> — Hôte auquel se lier ; par défaut, utilise la valeur gateway.host de la configuration. Remarque : La liaison à 0.0.0.0 nécessite gateway.allow_public_bind = true dans la configuration.
  • --allow-degraded-security — Démarrer même lorsque des sections de configuration critiques pour la sécurité ont été ramenées à leurs valeurs par défaut lors du chargement (posture de sécurité affaiblie). Désactivé par défaut

zeroclaw gateway restart

Redémarrez le serveur de passerelle.

Arrête le gateway en cours d’exécution, s’il est présent, puis démarre une nouvelle instance avec la configuration actuelle.

Exemples : zeroclaw gateway restart # redémarrer avec les valeurs par défaut de la configuration zeroclaw gateway restart -p 8080 # redémarrer sur le port 8080

Utilisation : zeroclaw gateway restart [OPTIONS]

Options :

  • -p, --port <PORT> — Port d’écoute (utilisez 0 pour un port aléatoire disponible) ; par défaut, utilise la valeur de gateway.port dans la configuration.
  • --host <HOST> — Hôte auquel se lier ; par défaut, utilise la valeur gateway.host de la configuration. Remarque : La liaison à 0.0.0.0 nécessite gateway.allow_public_bind = true dans la configuration.
  • --allow-degraded-security — Démarrer même lorsque des sections de configuration critiques pour la sécurité ont été ramenées à leurs valeurs par défaut lors du chargement (posture de sécurité affaiblie). Désactivé par défaut

zeroclaw gateway get-paircode

Afficher ou générer le code d’appairage de la passerelle.

Affiche le code d’appairage pour connecter de nouveaux clients sans redémarrer la passerelle. La passerelle doit être en cours d’exécution.

Avec –new, génère un nouveau code d’appairage même si la passerelle a déjà été appairée (utile pour ajouter des clients supplémentaires). Cela ne révoque PAS les jetons existants.

Avec –rotate, révoque TOUS les jetons bearer appairés, efface le registre des appareils et émet un nouveau code. Utilisez cette option après une fuite de jeton suspectée lorsque vous ne savez pas quel jeton a été compromis ; chaque client doit se réappairer.

With –rotate-device ID, revokes just that device’s bearer token and issues a fresh code for re-pairing that one device.

Exemples : zeroclaw gateway get-paircode # afficher le code d’appairage actuel zeroclaw gateway get-paircode –new # ajouter un autre client (sans révocation) zeroclaw gateway get-paircode –rotate # révoquer TOUS les jetons, puis émettre un code zeroclaw gateway get-paircode –rotate-device dash-1 # révoquer le jeton d’un appareil zeroclaw gateway get-paircode –new –port 3001 # cibler la passerelle sur un port alternatif

Utilisation : zeroclaw gateway get-paircode [OPTIONS]

Options :

  • --new — Génère un nouveau code d’appairage pour ajouter un client (ne révoque pas les jetons existants)
  • --rotate — Révoque TOUS les jetons appairés et efface le registre des appareils, puis émet un nouveau code
  • --rotate-device <DEVICE_ID> — Révoque le jeton porteur d’un seul appareil par identifiant, puis émet un nouveau code
  • -p, --port <PORT> — Port de la passerelle en cours d’exécution à interroger ; par défaut config gateway.port
  • --host <HOST> — Hôte de la passerelle en cours d’exécution à interroger ; par défaut config gateway.host

zeroclaw acp

zeroclaw daemon

Démarrer le démon autonome à longue exécution.

Démarre l’exécution complète du runtime ZeroClaw : serveur gateway, tous les canaux configurés (Telegram, Discord, Slack, etc.), moniteur de battement de cœur et planificateur cron. C’est la méthode recommandée pour exécuter ZeroClaw en production ou en tant qu’assistant toujours actif.

Utilisez ‘zeroclaw service install’ pour enregistrer le daemon en tant que service du système d’exploitation (systemd/launchd) afin qu’il se lance automatiquement au démarrage.

Exemples : zeroclaw daemon # utiliser les valeurs par défaut de la configuration zeroclaw daemon -p 9090 # passerelle sur le port 9090 zeroclaw daemon –host 127.0.0.1 # uniquement localhost

Utilisation : zeroclaw daemon [OPTIONS]

Options :

  • -p, --port <PORT> — Port d’écoute (utilisez 0 pour un port aléatoire disponible) ; par défaut, utilise la valeur de gateway.port dans la configuration.
  • --host <HOST> — Hôte auquel se lier ; par défaut, utilise la valeur gateway.host de la configuration
  • --ephemeral — S’arrête automatiquement après la déconnexion de tous les clients socket (avec un délai de grâce)
  • --allow-degraded-security — Démarre même lorsque des sections de configuration critiques pour la sécurité ont été réinitialisées à leurs valeurs par défaut lors du chargement. Sans cette option, le démon refuse de démarrer avec une posture affaiblie ; avec elle, le démon démarre afin que l’opérateur puisse accéder aux interfaces de réparation, en émettant un avertissement répété

zeroclaw cron

Configurer et gérer les tâches planifiées.

Planifiez des tâches récurrentes, ponctuelles ou à intervalles à l’aide d’expressions cron, d’horodatages RFC3339 avec Z explicite ou décalages, de durées ou d’intervalles fixes.

Les expressions cron utilisent le format standard à 5 champs : ‘min hour day month weekday’. Lorsque –tz est omis, les planifications cron utilisent le fuseau horaire local de l’environnement d’exécution. Pour les planifications destinées aux utilisateurs, passez –tz avec un fuseau horaire IANA explicite.

Examples: zeroclaw cron list zeroclaw cron add ‘0 9 * * 1-5’ ‘Good morning’ –tz America/New_York –agent zeroclaw cron add ‘*/30 * * * ’ ‘Check system health’ –agent zeroclaw cron add ‘/5 * * * *’ ‘echo ok’ zeroclaw cron add-at 2025-01-15T14:00:00Z ‘Send reminder’ –agent zeroclaw cron add-every 60000 ‘Ping heartbeat’ zeroclaw cron once 30m ‘Run backup in 30 minutes’ –agent zeroclaw cron pause TASK_ID zeroclaw cron update TASK_ID –expression ‘0 8 * * *’ –tz Europe/London

Utilisation : zeroclaw cron <COMMAND>

Sous-commandes :

  • list — Afficher toutes les tâches planifiées
  • add — Ajouter une nouvelle tâche planifiée
  • add-at — Ajouter une tâche planifiée à exécution unique à un horodatage RFC3339 avec Z explicite ou décalage
  • add-every — Ajouter une tâche planifiée à intervalle fixe
  • once — Ajouter une tâche retardée à exécution unique (par exemple, “30m”, “2h”, “1d”)
  • remove — Supprimer une tâche planifiée
  • update — Mettre à jour une tâche planifiée
  • pause — Mettre en pause une tâche planifiée
  • resume — Reprendre une tâche en pause

Exemples : zeroclaw cron add ‘0 9 * * 1-5’ ‘Bonjour’ –tz America/New_York –agent zeroclaw cron add ‘*/30 * * * ’ ‘Vérifier la santé du système’ –agent zeroclaw cron add ‘/5 * * * *’ ‘echo ok’

Utilisez --check pour vérifier les mises à jour sans les installer. Utilisez --force pour ignorer l’invite de confirmation. Utilisez --version pour cibler une version spécifique au lieu de la dernière.

Exemples : zeroclaw update # télécharger et installer la dernière mise à jour de zeroclaw zeroclaw update –check # vérifier uniquement, ne pas installer zeroclaw update –force # installer sans confirmation zeroclaw update –version 0.6.0 # installer une version spécifique

Utilisation : zeroclaw update [OPTIONS]

Options :

  • --check — Vérifier uniquement les mises à jour, sans installer
  • --force — Install even if the target is not newer (reinstall or downgrade/pin to –version)
  • --version <VERSION> — Version cible (par défaut : dernière version)

zeroclaw locales

Fetch translated Fluent (.ftl) catalogues for a locale from the upstream repository and install them under <config-dir>/data/ftl/<locale>/, where the runtime and zerocode loaders read them.

Transmettez une seule locale. Par défaut, tous les catalogues sont récupérés ; restreignez avec –catalog (séparés par des virgules) : cli, tools, zerocode.

Exemples : zeroclaw locales fetch ja zeroclaw locales fetch fr –catalog cli,tools zeroclaw locales fetch zh-CN –catalog zerocode

Utilisation : zeroclaw locales <COMMAND>

Sous-commandes :

  • fetch — Télécharger les fichiers FTL traduits pour une locale depuis le dépôt amont
