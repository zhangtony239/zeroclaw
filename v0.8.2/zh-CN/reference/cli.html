zeroclaw

最快、最小的 AI 助手。

用法： zeroclaw [OPTIONS] <COMMAND>

子命令：

  • quickstart — Quickstart — create one working agent end-to-end. Replaces the section-by-section onboarding flow with a single preset-driven path. Interactive: the flags below pre-seed checklist selectors but do not skip them; a terminal is required
  • onboard — 已弃用。请使用 zeroclaw quickstart。任何标志都将报错
  • agent — 启动 AI 代理循环
  • gateway — 启动/管理网关服务器（webhooks、websockets）
  • acp — 通过 stdio 启动 ACP（Agent Control Protocol）服务器
  • daemon — 启动长期运行的自主运行时（网关 + 通道 + 心跳 + 调度器）
  • service — 管理操作系统服务生命周期（launchd/systemd 用户服务）
  • doctor — 运行守护进程/调度器/通道新鲜度的诊断
  • status — 显示系统状态（详细信息）
  • estop — 启用、检查并恢复紧急停止状态
  • cron — 配置和管理定时任务
  • models — 管理 model_provider 模型目录
  • providers — List supported AI model providers, or manage provider aliases
  • channel — 管理频道（Telegram、Discord、Slack）
  • agents — Manage agent aliases (create/list/rename/delete). Distinct from agent, which runs an agent
  • channels — Manage channel aliases (create/list/rename/delete)
  • integrations — 浏览 50 多种集成
  • skills — 管理技能（用户定义的能力）
  • browse — 逐个目录浏览共享工作区
  • sop — 管理标准操作程序（SOP）
  • migrate — 从其他代理运行时迁移数据
  • auth — 管理 model_provider 订阅认证配置文件
  • hardware — 发现并检查 USB 硬件
  • peripheral — 管理硬件外设（STM32、RPi GPIO 等）
  • memory — 管理代理记忆（列出、获取、统计、清除）
  • config — 管理配置
  • update — 检查并应用更新
  • self-test — 运行诊断自检
  • completions — 生成 shell 补全脚本并输出到标准输出
  • desktop — 启动或安装配套桌面应用
  • locales — 从上游获取已翻译的语言区域文件（FTL）

选项：

  • --config-dir <CONFIG_DIR>

  • --log-level <LOG_LEVEL> — 记录到运行时跟踪（及捕获层）的最低严重级别。在进程生命周期内不可变更。优先级顺序：此标志 > RUST_LOG 环境变量 > 各命令的默认值

    可选值：error、warn、info、debug、trace

  • -v、--verbose — 在终端显示已记录的日志。默认关闭：日志仅写入跟踪文件，终端只显示命令输出。开启时，终端会显示低至记录下限的事件。在进程运行期间不可更改

zeroclaw quickstart

Quickstart — create one working agent end-to-end. Replaces the section-by-section onboarding flow with a single preset-driven path. Interactive: the flags below pre-seed checklist selectors but do not skip them; a terminal is required

用法： zeroclaw quickstart [OPTIONS]

选项：

  • --model-provider <MODEL_PROVIDER> — 提供商类型（anthropic / openai / openrouter / ollama）
  • --model <MODEL> — 新提供方条目的模型 id
  • --api-key <API_KEY> — 新提供商条目的 API 密钥（ollama / 本地时可省略）

zeroclaw onboard

已弃用。请使用 zeroclaw quickstart。任何标志均会报错

用法： zeroclaw onboard [COMMAND]

子命令：

  • providers.models — 选择要配置的模型提供商（Anthropic、OpenAI、OpenRouter、Ollama、自定义 OpenAI 兼容网关等）。每个提供商支持配置多个别名——例如 anthropic.production 和 anthropic.dev 可以并存
  • risk_profiles — 命名的风险配置文件，用于绑定允许列表、拒绝列表和审批阈值。代理可通过 agents.<alias>.risk_profile 引用其中一个
  • runtime_profiles — 命名的运行时调优配置文件（令牌限制、重试策略、超时设置）。智能体通过 agents.<alias>.runtime_profile 引用其中之一
  • storage — SQLite 是单节点安装时的安全默认选项（基于文件、零配置、无需额外服务）。共享或多实例部署请选用 Postgres，向量检索请选用 Qdrant，需要人类可读文件时请选用 Markdown 或 Lucid。每种后端都支持多个带别名的实例；代理通过 memory.storage_ref 引用它们
  • memory — 持久化内存后端。默认使用 SQLite；选择 none 可完全禁用长期记忆
  • skills — 技能工具设置 — 技能 markdown 文件在磁盘上的存放位置（默认为数据目录），以及技能加载器如何处理社区仓库。请在下方的 skill-bundles 下添加技能 BUNDLES
  • skill_bundles — 技能文件的命名捆绑包。智能体通过引用捆绑包，在启动时加载一组功能
  • mcp — 模型上下文协议（Model Context Protocol）设置。切换 enabled 并选择延迟加载或预先加载。各个 MCP 服务器配置位于 mcp.servers[] 下
  • mcp.servers — Individual Model Context Protocol servers. Each entry binds a transport (stdio, http, sse), the command or URL to reach it, optional headers, and a tool_timeout_secs cap (≤ 600). Each server’s name is its addressable key — rename via the section page rather than editing the field directly. Group servers into bundles under mcp_bundles below
  • mcp_bundles — Named bundles of MCP servers, granted to agents that list the bundle in their mcp_bundles. Secure by default: an agent gets only the servers its bundles grant; with no bundle it gets no MCP servers
  • knowledge_bundles — 知识源的命名捆绑包（RAG 索引、文档文件夹）。Agent 引用某个捆绑包，以便在推理时呈现相关的代码片段
  • providers.tts — 文本转语音提供商（OpenAI、ElevenLabs、Google、Edge、Piper）。为每种语音/语言配置一个提供商；代理通过别名引用它们
  • providers.transcription — 语音转文字提供商（OpenAI Whisper、Groq、Deepgram、AssemblyAI、Google、本地 Whisper）。每个流水线配置一个；代理通过别名引用它们
  • channels — Pick which chat platforms ZeroClaw should listen on. Global channel settings live on [channels]; each configured platform still gets its own alias
  • hardware — 可选：硬件外设（Arduino、STM32、GPIO 等）。如不需要可跳过
  • agents — 智能体将模型提供方、配置文件、捆绑包和通道绑定为一个可调度的单元。每个角色添加一个智能体；在多个通道中复用同一个别名以共享状态
  • peer_groups — 绑定一个通道、成员代理和外部对等方的命名组。需双方同意：只有当两个代理都出现在同一组的 agents 列表中时，它们才会成为对等方
  • cron — 定时任务。每个 cron 条目将调度表达式绑定到一个提示词、通道和目标
  • tunnel — 可选：通过 Cloudflare 或 ngrok 将网关暴露到公共互联网。选择 none 可仅保留本地访问
  • onboard_state — Quickstart 生命周期状态。quickstart_completed 在 Quickstart 成功完成一次运行后置为 true；当为 false 时，Web 网关和 TUI 会在启动时自动启动 Quickstart。completed_sections 是一个旧版的逐节记录，保留用于与先前数据的向后兼容

zeroclaw agent

启动 AI 代理循环。

启动与已配置 AI 模型提供方的交互式聊天会话。使用 –message 可进行单次查询，无需进入交互模式。

zeroclaw gateway start

启动网关服务器（webhooks、websockets）。

运行 HTTP/WebSocket 网关，该网关接受传入的 webhook 事件和 WebSocket 连接。绑定地址默认为配置文件中的值（gateway.host / gateway.port）。

示例：zeroclaw gateway start # 使用配置默认值 zeroclaw gateway start -p 8080 # 监听端口 8080 zeroclaw gateway start –host 0.0.0.0 # 需要 [gateway].allow_public_bind=true 或隧道 zeroclaw gateway start -p 0 # 随机可用端口

用法： zeroclaw gateway start [OPTIONS]

选项：

  • -p, --port <PORT> — 监听的端口（使用 0 表示随机可用端口）；默认为配置项 gateway.port
  • --host <HOST> — 要绑定的主机；默认为配置中的 gateway.host。注意：绑定到 0.0.0.0 需要在配置中设置 gateway.allow_public_bind = true
  • --allow-degraded-security — 即使在加载期间将安全关键的配置部分还原为默认值（削弱安全态势），也继续启动。默认关闭

zeroclaw gateway restart

重启网关服务器。

如果存在正在运行的网关，则停止它，然后使用当前配置启动新实例。

示例：zeroclaw gateway restart # 使用配置默认值重启 zeroclaw gateway restart -p 8080 # 在端口 8080 上重启

用法： zeroclaw gateway restart [OPTIONS]

选项：

  • -p, --port <PORT> — 监听的端口（使用 0 表示随机可用端口）；默认为配置项 gateway.port
  • --host <HOST> — 要绑定的主机；默认为配置中的 gateway.host。注意：绑定到 0.0.0.0 需要在配置中设置 gateway.allow_public_bind = true
  • --allow-degraded-security — 即使在加载期间将安全关键的配置部分还原为默认值（削弱安全态势），也继续启动。默认关闭

zeroclaw gateway get-paircode

显示或生成网关配对码。

显示用于连接新客户端的配对码，而无需重启网关。需要网关正在运行。

使用 --new 时，即使网关之前已配对，也会生成新的配对码（适用于添加额外的客户端）。此操作不会撤销现有的令牌。

使用 –rotate 时，将吊销所有已配对的持有者令牌，清除设备注册表，并签发新代码。在怀疑令牌泄露但不确定哪个令牌已被泄露时使用此选项；所有客户端都必须重新配对。

With –rotate-device ID, revokes just that device’s bearer token and issues a fresh code for re-pairing that one device.
