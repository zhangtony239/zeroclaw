ZeroClaw Docs

快速开始

Quickstart 是引导式安装流程，可一次性将你从全新安装带到可用的智能体。它运行于三种环境：CLI、zerocode 终端界面以及 web gateway。这三者驱动相同的底层流程，因此生成的配置完全一致。可根据你所处的环境选用任意一种。

安装

Linux

一键安装（通过 curl 运行 install.sh）：

curl -fsSL https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/install.sh | sh

从克隆仓库：

./install.sh

Homebrew (Linuxbrew):

brew install zeroclaw

macOS

一键安装（通过 curl 运行 install.sh）：

curl -fsSL https://raw.githubusercontent.com/zeroclaw-labs/zeroclaw/master/install.sh | sh

从克隆仓库：

./install.sh

Homebrew：

brew install zeroclaw

Windows

setup.bat（来自发布版本）：

setup.bat

Scoop：

scoop install zeroclaw

来源：

cargo install --locked --path .

在 WSL2 上，请按照 Linux 路径操作；install.sh 无需修改即可运行。完整的操作步骤请参阅 Setup → Windows。

这会构建并安装 zeroclaw 和 zerocode 终端界面。不带任何标志运行它，即可进入交互式选择器，让你选择构建类型、要安装的应用以及要编译进来的可选功能。

步骤

重要： 如果你对其中任何术语不熟悉，请先阅读 Getting Started → Concepts。它在一处定义了 model provider、risk profile、alias 以及其余术语。

每个界面都遵循相同的检查清单并写入相同的配置。必需步骤必须满足后才能创建该代理；可选步骤可以跳过。

步骤必填设置内容
模型提供商是提供商系列（Anthropic、OpenAI、Ollama、OpenRouter 等）、其 API 密钥或端点以及模型。
风险概况是自主性与沙箱配置。选择一个预设或复用现有的 [risk_profiles.<alias>]。
内存是内存后端（sqlite、markdown、postgres、qdrant、lucid 或 none）。
通道可选聊天平台（Telegram、Discord、Slack 等）。内置的 cli 通道始终可用；可在此处或稍后添加其他平台。
对等组可选您配置的频道的多代理对等成员资格。
Agent是Agent 别名、系统提示词以及任何个性化文件。

运行时配置文件会自动设置。Quickstart 会为新代理安装 unbounded 预设。之后可通过编辑 [runtime_profiles.<alias>] 来调整预算和超时设置（参见 Reference → Config）。

命令行界面

在无头机器上或通过 SSH 操作时最快的路径：

sh

zeroclaw quickstart

你可以在终端中每步回答一个提示。内置的 cli 通道可立即使用，因此可以跳过 Channels 和 Peer groups。如需全部使用默认值、无需审批的配置，请参阅 YOLO mode。

zerocode

在 zerocode 终端界面中，Quickstart 窗格是其中一个选项卡。使用键盘来操作它：

切换到 Quickstart 窗格：

密钥操作
Alt+← / Alt+b上一个窗格
Alt+→ / Alt+f下一个窗格

窗格内：

密钥操作
k / ↑prev
j / ↓下一个
Enter打开
q / Esc离开
c / C创建代理

鼠标同样可用：单击模式栏中的标签页可切换窗格，单击某个步骤可选中并打开它，滚动则可在列表中移动浏览。

每个步骤都会打开一个与上述清单相对应的模态框，其中包含一个“使用现有”选项，该选项会列出配置中已匹配的别名。

Web 网关

在守护进程运行的情况下，在浏览器中打开仪表板：

sh

zeroclaw daemon

zeroclaw daemon 运行完整的运行时：网关、你配置的通道、调度器以及心跳监视器。（如果你只需要 HTTP 网关，可使用 zeroclaw gateway 仅启动它。）

然后访问 http://127.0.0.1:42617/quickstart。全新安装且未配置任何 agent 时会自动重定向到该页面；之后你随时可以从仪表板导航中访问它。

Web 表单以卡片形式呈现相同的步骤。提交时，它会通过守护进程应用你的提交内容（POST /api/quickstart/apply），如果有任何无效内容，将返回结构化的错误列表，然后就地重新加载守护进程，使新代理无需重启即可生效。另有一个独立的 POST /api/quickstart/validate 端点，可在不应用的情况下运行相同的检查，供希望先验证的客户端使用。

快速入门后

  • 通过 zerocode 操作： 终端界面是聊天、查看实时日志、管理配置和监控守护进程的最佳方式，所有功能集于一处。只需运行 zerocode。
  • 从 shell 快速执行一次性操作： zeroclaw agent -a <alias> -m "your message"。
  • 始终运行： zeroclaw service install && zeroclaw service start。
  • 稍后添加渠道： 渠道 → 概述。
  • 调整自主性和预算： 参考 → 配置。
